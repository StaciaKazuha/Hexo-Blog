前置知识

视频编码

由于未压缩的视频数据的使用巨大的空间，有必要对其进行压缩以便进行存储。

目前主流的编码标准是 HEVC（H.265）和 AVC（H.264），这两种视频编码受大量设备广泛支持。

而标准之下由对应的编解码器，例如，x264，它是 AVC/H.264 的实现，以及 x265，它是 H.265/HEVC 的实现。

音频编码

在电影中，音频编码格式 主要分为 DVD 时代的有损编码 和 次世代的无损编码

其中，DVD时代有 DTS，AC3，AAC 三种格式。而次世代有 DTS-HD，TrueHD。

而次世代音轨为了兼容旧设备的播放，往往会附带一个内核音频，也就是其有损编码版本。

DTS-HD 对应的是 DTS，TrueHD 对应的是 AC3。

安装软件

解压软件

认证软件

运行StoneStudio_MovieInfo.exe，第一次运行会显示未填写设置，先不管他。

第一次打开还会看到未授权。

打开 设置。
如果你没有使用 Edge 浏览器，请点击 安装EdgeWebView2。
接着点击 登录，输入你的 HDSky 账号和密码，认证软件。
在设置页面，调整封装路径和做种路径。
封装路径：点击封装功能后，影片输出的文件夹
做种路径：点击一键发种后，影片移动到 QB 做种目录对应的文件夹
最后点击保存即可，接着重启软件！

配置软件环境

如果你注册软件成功，就可以看见这个选项卡，否则这个选项卡是看不见的。请点击 压制环境选项 。

  1. 在弹出窗口，点击 配置Python 。请 Install for all users
  2. 打勾 Add to Path, 点击 Customize installation -> 打勾所有选项 -> 打勾 Install for all users -> 安装即可。
  3. 安装完 Python 就点击第二个按钮，安装 VapourSynth ，请 Install for all users 。建议安装里面所有组件。
  4. 如果你电脑里没有装过Java，建议安装 Java Runtime ，否则 BDSup2Sub 之类的工具就没法使用。
  5. 点击配置 VS Plugins ，这步无需操作，等待执行完毕显示 请按任意键继续 即可。如果显示任何出错的地方，请重新执行配置 VS Plugins ，个人建议多运行几次。
  6. 修复语法高亮：如果编辑 VS 脚本没有语法高亮，就在这里点修复一下。这个不是必须的。

下载原盘

下载到的原盘可能是 文件夹 格式的，也可能是 ISO 格式的，如果是 ISO，可虚拟成光驱或解压成文件夹。

上图是 ISO 文件，下图是蓝光文件夹

确认文件名

把原盘的名称复制到原盘来源中，点击 从原盘获取信息 ，会自动分析，并给出影片命名。

因为不能保证信息完全正确，所以需要人工检查一遍。

Tip：个人习惯，会去掉前面的中文。

制作 1080P REMUX

制作 Remux 需要使用到两个工具，BDInfo 和 Mkvtoolnix。

找到 mpls 文件

打开 BDInfo ，选择蓝光文件夹，会扫出播放列表，核对对应长度的播放列表以及对应的轨道。
如图，00020.MPLS 为正片的播放列表。

选择所需轨道并导出

把 00020.MPLS 拖入到 Mkvtoolnix 中，会显示出所有的轨道。

选择需要保留的轨道，不需要的轨道去掉勾选。

保留思路如下：

  1. 音轨方面，保留影片的原声音轨和中文音轨，
    如果有多个相同语言的音轨，保留质量最好的那一个。
    先看格式 TrueHD Atmos > TrueHD 7.1 > TrueHD 5.1 > DTS-HD > LPCM
    相同格式选码率高者
    如果勾选 TrueHD 轨道，需要同时勾选下面的 AC3 内核轨道。

  2. 字幕方面，保留原声语言字幕和中文简繁字字幕

并选择目标文件夹。创建影片的文件夹，把 remux.mkv 文件输出到这里
按 开始混流 。
完成之后你就拿到了 Remux 可以用于后续压制了。

制作 4K REMUX

制作 Remux 需要使用到 MakeMKV

选择 BDMV 文件

打开 MakeMKV ，选择蓝光文件夹的 bdmv 文件。

选择所需轨道并导出

保留正确的播放列表，这个应该时间与完整视频一致，并且有章节

保留思路如下：

  1. 音轨方面，保留影片的原声音轨和中文音轨，
    如果有多个相同语言的音轨，保留质量最好的那一个。
    先看格式 TrueHD Atmos > TrueHD 7.1 > TrueHD 5.1 > DTS-HD > LPCM
    相同格式选码率高者
    如果勾选 TrueHD 轨道，需要同时勾选下面的 AC3 内核轨道。
    如果勾选 DTS-HD 轨道，需要同时勾选下面的 DTS 内核轨道。

  2. 字幕方面，保留原声语言字幕和中文简繁字字幕
    对于一个字幕，需要勾选 字幕 和 强制字幕。

并选择目标文件夹。创建影片的文件夹，把 remux.mkv 文件输出到这里。
按 开始 。
完成之后你就拿到了 Remux 可以用于后续压制了。

制作 dgi

注意：DGNV要求必须使用NVIDIA显卡。如果没有，你可以咸鱼弄一块低端的 N卡，比如说 GTX960 之类的装进机器里面作为第二块显卡使用。

  1. 启动 DGNV
  2. 把影片文件文件拖进 DGIndexNV 中
  3. 选择 File 菜单中的 Save Project ，保存一个DGI的工程文件，等待提示 Finished 即可。

注意这个时候，除了视频轨，目录里面还生成了若干音频，字幕和章节。这些轨道保存备用。

生成 AAC 音频

打开 StoneStudio_MovieInfo.exe，选择对应的音频，勾选导出WAV，开始导出！

压制视频轨

打开 StoneStudio_EncoderTool.exe ，选择正确的模式，选择导出的 dgi 文件，单击 VS 脚本编辑，检查填写是否正确。

如果你想压制 PAD 视频，选择 HEVC PAD 模式，选择 字幕文件，单击 VS 脚本编辑，检查填写是否正确。

如果你想压制 DV 视频，选择 4K HEVC DV 模式，选择 字幕文件，选择 dv rpu 文件，单击 VS 脚本编辑，检查填写是否正确。

压片分边两步，
首先，压制 10% 样片来确定码率是否合理，这个步骤非常简单，直接点“样片试压“。
会自动帮你运行批处理，请参考文档 码率和封装说明 进行调整。码率的调整可以通过批处理中的CRF值来调整，CRF值越大码率越低，CRF值越小码率越大。可以多压几次来控制。
通常来讲，样片压制的码率会比最终生成的码率略大2%-5%。仅供参考。
样片压制没问题，就直接点“批量压制“功能，会自动帮你输出成片，无需你控制。

样片码率可以在 EncodingArtifactsExport 文件夹下 找到 Sample 前缀的 log 文件查看。

完成后，在 EncodingArtifactsExport 文件夹下能找到相应的视频文件

同名还有一个 log 文件，这个在后面生成详情的时候有用到。

X264 X265，压制方法一样，以此类推，不用设置字幕，具体要求和参数见 码率和封装说明

合并封装 mkv

请确认 影片命名 填写正确，封装将按照其命名导出。

按照影片轨道填入信息并调整。

若用 DGNV 导出轨道，章节文件选择不带 qp 结尾的文件。

上传种子

最后一步啦！

先打开封装产物，使用 Potplater，按 alt+n 生成九宫格图片，具体配置如下。

打开一键发种页面，填入各项位置。

对于重编码影片来说，需要 勾选对比图 和 放压制日志。
存在特例，如果 x264 的 QP 大于 21 的影片，则不需要 勾选对比图 和 放压制日志。

单击获取截图后，挑选并删除不必要的截图后，单击上传生成 BBCode

单击本片豆瓣 url 右边三点，跳转后打开对应影片的链接，复制对应链接填入。

码率和封装说明

1080P 全家桶 为 REMUX+264+265+Pad
4K SDR 是 REMUX+265
4K HDR 或 DV 是 REMUX+pad+265

  1. 轨道选择

    • 视频轨道：选择1080P轨道，其他视频轨道都不要。
    • 音频轨道：选择原版音轨，国语，粤语音轨。不再需要提取内核。
    • 字幕轨道：选择原片字幕，简体中文，繁体中文，简英双字，繁英双字。

  2. 封装（音轨说明）

    • REMUX的音轨：REMUX中封装的音频，格式必须与原盘完全相同，如原盘是DTS-MA 7.1，REMUX也必须相同。
    • PAD音轨：必须编码成AAC格式，最多可以封装4条音轨。
    • 重编影片的音轨封装，选择原版音轨，国语，粤语音轨，最多可以封装4条音轨。

  3. 码率

    • 重编码x264：码率在7000-14000Kbps之间，并令QP尽量小于并接受18.00，可以LOG文件中查看
    • 重编码x265：以重编码x264的码率为参考的 60%-80%，即码率在4000-10000Kbps之间
    • Pad：1.7Mbps-2.3Mbps 之间。
    • 4K：大片 20Mbps-30Mbps，非大片 17Mbps-23Mbps
    • 4KPad：4Mbps-5Mbps 之间。
    • Dolby Vision和HDR10 Plus的码率说明：
      • 其中DV相比4K多给3Mbps。例如，大片23Mbps-33Mbps，非大片20Mbps-26Mbps，下同。
      • HDR10+相比4K多给1Mbps。双修相比4K多给4Mbps。
